第 1 章 预备知识

1.2 C++ 简介和简史

C 语言：自顶向下结构化编程
C++：融合了OOP、泛型编程和传统的过程性方法

1.2.3 面向对象编程OOP

类是描述新型数据格式的一种规范，对象是根据这种规范构造的特定数据结构。

优点：

  • 有助于创建可重用的代码，减少大量的工作；
  • 信息隐藏可以保护数据，使其免遭不适当的访问；
  • 多态让我们能为运算符和函数创建多个定义，通过编程上下文来确定使用哪个定义；
  • 继承允许使用旧类派生出新类

bottom-up编程
OOP程序设计方法首先设计类来表示程序要处理的东西，类规定了可使用哪些数据来表示对象以及可以对这些对象进行哪些操作，也就是将数据和方法合并为类定义。然后可以设计一个使用这些类的对象的程序。从低级组织（如类）到高级组织（如程序）的处理过程叫做自下向上（bottom-up）的编程。

1.2.4 泛型编程

泛型编程 (generic programming) 是C++支持的另一种编程模式。“泛型”指的是创建独立于类型的代码，C++的数据表示有很多类型——整数、小数、字符、字符串、用户定义的、由多种类型组成的复合结构。如果要想对不同类型的数据进行排序，通常必须为每种类型创建一个排序函数。泛型编程需要对语言进行扩展，以便可以只编写一个泛型（即不是特定类型的）函数，并将其用于各种实际类型。

它和OOP的目标相同，都是为了使重用代码和抽象通用概念的技术更简单。不同点在于：

  • OOP强调的是编程的数据方面，而泛型强调的是独立于特定数据类型；
  • OOP是一个管理大型项目的工具，而泛型编程提供了执行常见任务（如对数据排序或合并链表的工具）

2.1 进入C++

2.1.1 main() 函数

C++函数可以被其他函数激活或调用，函数头描述了函数与调用它的函数之间的接口，函数名后括号中的部分叫 形参列表(argument list) 或 参数列表(parameter list)。

main() 在启动代码时被调用，启动代码是由编译器添加到程序中的，是程序和操作系统之间的桥梁，实际上该函数头描述的是main()和操作系统之间的接口。可以将操作系统看作调用程序，它们使用并测试返回值（通常叫做退出值），退出值为0表示程序运行成功，非零则意味着有问题。

main()函数可以给调用它的函数返回一个整数值，且不从调用它的函数那里获得任何信息。

在运行C++程序时，通常从main()函数开始执行。例外情况：

  • Windows 编程中，可以编写一个动态链接库 (DLL) 模块，是其他Windows程序可以使用的模块，由于DLL模块不是独立的程序，因此不需要main()；
  • 用于专用环境的程序可能不需要main，如机器人中的控制芯片。

2.1.4 头文件名

像iostream这样的文件叫包含文件（include file）——因为他们被包含在其他文件中；也叫头文件（header file）——因为他们被放在文件起始处。

C++编译器自带了很多头文件，每个头文件都支持一组特定的工具，老式C的头文件保留了扩展名h，C++头文件没有扩展名，有些C头文件被转换为C++头文件，这些文件被重新命名，去掉了扩展名h，并在文件名称前面加上前缀c，如C++版本的math.h为cmath。没有h的头文件也可以包含名称空间。

2.1.5 名称空间

如果使用iostream，而不是iostream.h，则应使用名称空间编译指令using namespace std;来使iostream中的定义对程序可用。这条指令叫using编译指令。

名称空间支持时一项C++特性，目的是让用户在编写大型程序或将多个厂商现有的代码组合起来的程序时更容易，还有助于组织程序。比如有两个已经封装好的产品，它们都包含了一个叫wanda() 的函数，导致在使用wanda()函数时，编译器不知道指的是哪个版本。名称空间让厂商能够将其产品封装在一个叫做名称空间的单元中，这样就可以用名称空间的名称来指出想用哪个厂商的产品。

类、函数和变量是C++编译器的标准组件，头文件没有扩展名h，他们都被放在名称空间std中。这意味着在iostream中定义的用于输出的cout变量实际上是std::cout，而endl实际上是 std::endl，using namespace std; 表明可以使用std名称空间中定义的名称，不必用std::前缀，所以如果参略编译指令using，也可以用下面的代码进行编码：

<< 符号表示把这个字符串发送给cout，该符号指出了信息流动的路径。

控制符

endl和\n的一个差别时，endl确保程序继续运行前刷新输出，即将其立即显示在屏幕上，而用换行符\n在有些系统中有时可能在用户输入信息后才会出现提示。

2.3.3 类简介

类是OOP的核心概念之一，是用户定义的一种数据类型，类定义描述的是数据格式及其用法，就是它能够表示什么信息和对数据可以执行哪些操作；类的对象则是根据类的数据格式规范创建的实体，类之于对象就像类型之于变量。类描述了一种数据类型的全部属性（包括可使用它执行的操作），

ostream 和 istream类来自类库，从技术上说，类库的中的类没有被内置到C++语言中，而是语言标准指定的类。

要对特定对象执行类定义中允许的操作，需要诶对象发送一条消息，有两种方式：

  • 使用类方法，本质上就是函数调用
  • 重新定义运算符，cin 和 cout 就是重新定义 << 运算符将“显示消息”发送给cout。

2.4 函数

2. 4. 1 函数原型

要在函数使用之前提供函数原型double sqrt(double);之于函数就像变量声明之于变量——指出涉及的类型（参数是发送给函数的信息，返回值是从函数发送回去的值）。结尾的分号表明它是一条语句，所以是一个原型而不是函数头。

函数原型不等于函数定义，原型只表述函数接口，即发送给函数的信息和返回的信息；而定义包含了函数的代码。

函数是一个由出入他们的信息所指定的黑盒子，函数原型将这种观点诠释地淋漓尽致。

C++将库函数的原型和定义分开，库文件中包含了函数的编译代码，而头文件中包含了原型。

C++可以使用C库函数，要使用库函数，应当包含提供该函数原型的头文件，例如在程序中使用sqrt()时必须提供原型，可以用两种方法实现：

  • 在源代码文件中输入函数原型；

  • 包含头文件cmath，其中定义了原型（推荐方法）。对C++库中的每个函数，都在一个或多个头文件中提供了其原型。

    通常的做法是把原型放到main()函数定义前面

编译器倾向于给函数名添加下划线前缀——提示它们对程序具有最后的发言权。

2.4.2 函数变体

int rand(void)和 int rand() 表示不接受任何参数。有些语言中，有返回值的函数称为函数，没返回值的称为过程或子程序。C++中这两种变体都被称为函数。

2.4.3 定义函数

函数定义不允许嵌套：C++ 不允许将函数定义嵌套在另一个函数定义中，每个函数定义都是独立的，所有函数的创建都是平等的。

参数列表：指出了何种信息将被传递给函数；
函数类型：指出了返回值的类型。

关键字：是计算机语言中的词汇，如int, void, return 和 double；main不是关键字，因为它不是语言的组成部分，但它是一个必不可少的函数的名称。
