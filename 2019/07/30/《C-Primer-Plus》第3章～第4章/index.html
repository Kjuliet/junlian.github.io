第 3 章

OOP 的本质是设计并扩展自己的数据类型，设计自己的数据类型就是让类型与数据匹配，如果做到这一点，以后使用数据时会发现容易得多。而C++ 内置的类型是在创建自己类型时的基本组件。

内置的类型分两组：

  • 基本类型：整数和浮点数，以及它们的多种变体；
  • 复合类型：在基本类型的基础上创建的复合类型，包括数组、字符串、指针和结构。

有了数据，还需要一种标识存储数据的方法，即变量

3.1 简单变量

为了把信息存储在计算机中，程序必须3个基本属性：

  • 存储在哪里；
  • 存储的是什么值；
  • 存储的是什么类型信息。

采取的措施就是声明一个变量，通过声明的类型描述了信息的类型，通过符号表示值的变量名，对int braincount; braincount = 5;两条命令， 程序将找到一块能够存储int类型的内存，将该内存单元标记为变量名braincount，然后将 5 复制到该内存单元中，之后就可以在程序中使用braincount来访问该内存单元。

可以用&运算符来检索braincount的内存地址

3.1.1 变量名

  • 以两个下划线或下划线加大写字母打头的名称被保留给编译器及其使用的资源使用；
  • 以一个下划线开头的名称被保留用作全局标识符。用_time_stop 这样的名称不会导致编译错误，但会导致行为的不确定性，换句话说，不知道结果将是什么。

命名方案：很多程序员会加入描述变量类型或内容的前缀，如strWeight, bWeight, pWeight, cWeight, intWeight 分别表示字符串、布尔、指针、字符、整数类型。

3.1.2 整型

宽度 (width)：买哦树存储整数时使用的内存量。基本整型按宽度递增的顺序摆列分别是 char、short、int、long、long long，每种类型都有符号版本和无符号版本，所以共有10种类型。char类型较特殊，常用来表示的是字符，而不是数字。

3.1.3 整型 short、int、long 和 long long

计算机内存的单元是位(bit)，short、int、long 和 long long 通过使用不同数目的位来存储值：

  • short 至少16位；
  • int 至少和short一样长；
  • long 至少32位，且至少和int一样长；
  • long long至少64位，且至少与long一样长。

位和字节
字节 (byte) 指 8 位 (bit) 的内存单元，8位内存块可以设置出256种不同的组合，所以 1 字节可以表示0-255或-128到127。
通常字节就是描述计算机内存量的度量单位，1KB = 1024字节，1MB = 1025KB。但C++对字节的定义不同，C++字节由至少能够容纳实现的基本字符集的相邻位组成，也就是说，可能取值的数目必须大于或等于字符数目。ASCII字符集可以用8位来容纳，而Unicode超过8位，所以有些实现可能使用16位甚至32位的字节。有些人使用术语八位组 (octet) 表示 8 位字节。

1. sizeof()运算符
用sizeof()运算符（运算符是内置的语言元素，对一个或多个数据进行运算，并生成 一个值）检查类型的长度，单位是字节。

2. 头文件 climits
头文件 climits 定义了符号常量 (用#define预处理方式) 来表示类型的限制，如：

  • CHAR_BIT：char 的位数
  • CHAR_MAX：char 的最大值
  • INT_MAX

#define 和 #include 一样也是预处理器编译指令，#define INT_MAX 32767上面的指令告诉预处理器，在程序中查找INT_MAX，把所有的 INT_MAX 都替换为 32767。

C++有更好的创建符号常量的方法：用关键字 const；但被设计成可用于C和C++的头文件中必须用#define。

3. C++11 初始化方式
初始化常规变量的方法与初始化类变量的方式更像。

3.1.4 无符号类型

short、int、long 和 long long都有不能存储负数值的无符号变体，优点是可以增大变量能够存储的最大值，当要存储人口、颗数等不可能为负的数值时使用：

带符号整型边界值+1 = 范围另一端的取值：
SHORT_MAX = 32767, SHORT_MAX + 1 = -32768。

3.1.6 整型字面值

即常量，如212 或 1776，C++用前两位来标识数字常量的基数：

  • 第一位为1～9，则基数为10（十进制）;
  • 第一位为0，第二位为1～7，则基数为8；
  • 前两位为 0x 或 0X，则基数为16。

cout 默认以十进制输出

3.1.7 确定常量的类型

如果有后缀名：

  • 后缀为 l 或 L：long；
  • 后缀为 u 或 U ：unsigned int；
  • 后缀为 ul 或 lu (大小写均可)：unsigned long；
  • 后缀为 ll 或 LL：long long.

如果不带后缀：

  • 十进制数，用int、long、longlong 中能够存储该数的最小类型表示；
  • 十六进制 / 八进制整数，用int、unsigned int、unsigned long、long long、unsigned long long中能存储该数的最小类型表示。因为十六进制常用来表示内存地址，而内存地址是没有符号的，所以unsigned int 比 long 更适合用来表示16位地址。
