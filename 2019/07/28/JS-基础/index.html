JS导入

行内导入：慎重，不安全 onclick=”alert(‘hello world!’)”
内嵌式导入： script> alert(‘hello world!’) </script>
外链式导入：script src=”js/import.js”> 内嵌导入和外链导入不能合并在一起，如果当前是外链导入，那么在脚本块中所有代码都不会被执行</script>

CSS导入

行内导入：div style=”” HTML和CSS混淆，代码结构混乱
内嵌导入：在head内写style> div{ height: 100px}
外链导入：新建css文件，文件内link rel=”stylesheet” href=”../..css”
导入式：在head->style块第一行写@import “css/…..css”
面试：扩展导入式和外链式的本质区别？

真实项目中，我们一般会把CSS放在BODY的上面，把JS放在BODY的末尾

BODY中编写的都是HTML标签，JS很多时候需要操作这些元素，首先我们要保证元素加载成功，才可以在JS中获取到（JS放在结构之后加载）

如果把JS放在HTML标签前面，要等结构加载完成才可以加载JS：
JS：window.onload=function(){} 页面中所有资源文件都加载完成再执行操作
JQ（JS中常用的库）：$(document).ready(function(){})页面中的结构加载完成就会执行操作，window.addEventListener(‘load’,function(){},false) / window.addEventListener(‘load’,function(){},false) / window.attachEvent(‘onreadyStatechange’,functiion(){})

浏览器弹框

  • alert：在浏览器中弹出一个提示框，默认确定按钮，点确定，弹框消失

    • alert(banbanFlag);

  • prompt：在confirm基础上增加让用户输入的效果

    • var flag = prompt(‘输入姓名:’);
      点取消flag=null,确定flag=输入内容，无输入内容，flag=””空字符串

      真实项目中，提示框一般都是自己封装插件和组件实现，不会用内置的（使用原声JS封装模态框组件）

JS的命名规范

  1. JS严格区分大小写

  2. 遵循国际命名规则：驼峰命名法

    第一个单词首字母小写，其余每个有意义单词首字母大写

    命名用英文单词，不要用拼音。
    不是每个单词都能简写，要保证看到名字后知道所代表的意思。
    info：infomation 信息
    init：initialization 初始化
    add / insert / create：增加创建插入
    remove / rm / del / delete：删除
    update：修改
    get / query / select：查询获取

  3. 命名可以使用$、_ 、数字，字母，但数字不能放第一位

  4. 不能用关键字和保留字

    关键字：有特殊含义的词
    保留字：现在没有特殊含义，以后可能会作为关键字的词

基本数据和引用类型的区别

JS是运行在浏览器中的（内核引擎）的，浏览器为JS提供执行环境：全局作用域window(global)

基本数据类型按值操作：在赋值的时候，直接把值赋值给变量即可。
引用数据类型是按空间地址（引用地址）操作的：var n = {name:’kk’}; 1.创建一个变量n；2. 浏览器首先会开辟一个新的存储空间，存储对象的键值对，为了方便找到这个空间，浏览器给空间设定了一个地址（16进制）；3. 把空间的地址赋值给变量。所以变量存储的仅仅是这个地址。

函数数据类型

函数数据类型也是按照引用地址来操作的

函数封装：把实现一个具体功能的代码封装在一个函数里。低耦合高内聚（没有重复的代码，减少冗余度。提高代码重复利用率）。

OOP面向对象编程的思想：继承、封装、多态

实际项目中，总结汇总，把公共部分提取，进行封装，方便之后复用。

核心原理：

  • 创建函数
  1. 首先在当前作用域中声明一个函数名，声明的函数名和使用var声明的变量名是一样的操作：var sum; function sum;两个名字算重复。
  2. 浏览器开辟一个新的内存空间，即分配一个16进制的地址，把函数体中写好的代码当作普通字符串存储在这个内存空间中，创建一个函数如果不执行，函数没有意义。
  3. 把内存空间地址赋值给之前声明的函数名。
  • 执行函数
  1. 浏览器首先为其开辟一个新的私有作用域，该作用域只能执行函数中之前编写的JS代码。
  2. 形参赋值
  3. 私有作用域的变量提升
  4. 把创建时存储的JS代码字符串拿到私有作用域，把它们变为JS表达式从上到下执行
  5. 私有作用域销毁or不销毁

闭包
函数执行时形成的私有作用域，里面的私有变量和外面的变量互不影响，相互不干扰，外面的无法直接获取里面的变量值，也就是把私有变量保护起来，这种保护机制称为闭包

栈内存
全局作用域或私有作用域，为JS代码执行提供环境

堆内存
所有的引用数据类型，它们需要存储的内容都在堆内存中，相当于一个仓库，对象会存储键值对，函数会把代码当作字符串存储。

形参
形参是个变量，提供了一个入口，在执行函数时可以把需要的值传递进来，形参用于存储和接受这些值。
实参
执行函数时传递的具体的值，如果没有值传递，为了保证结果不是NaN，需要为其设置默认值0

arguments实参集合
当不不知道用户具体要传递多少个值的时候，需要使用函数内置的实参集合：arguments

arguments只有函数才有
不管执行函数的时候是否传递实参，arguments天生就有，没传arguments为空的集合，有传则包含所有传递的实参
不管是否设置了形参，arguments始终存储了所有的实参信息。

arguments是一个类数组集合
1.以数字作为索引（属性名），从0开始，通过arguments[i]获取
2.有一个length的属性，存储当前传递实参的个数，arguments.length/arguments[‘length’]获取。
3.arguments.callee：存储的是当前函数本身（）
4.arguments.callee.caller：存储的是宿主函数，即当前函数是在哪执行的。在全局作用域下执性，结果是null

返回值 return

闭包的保护机制导致私有作用域会保护里面的变量，我们无法从外面直接获取里面的变量结果。返回值是函数提供的一个出口，让我们能够在外面使用函数的私有信息。

sum：代表函数本身。
sum()：代表函数返回的结果。

如果函数中没有return或只写return，默认返回结果为undefined
只写return：在函数体中遇到return，return后面的代码都不执行。

去重方法二:
用indexOf判断当前子数组中是否包含某一项，包含则把该项删除

去重方法三：对象去重法
遍历数组中每一项，把每一项作为新对象的属性名和属性值存储，每次存储之前验证当前对象中该属性是否存在（typeof obj[xxx] === 'undefined'说明当前对象中没有xxx属性），存在则不再进行存储并且把该项删除。

去重方法四：相邻比较法
排序，然后相邻比较，若重复，把后一项删掉。
