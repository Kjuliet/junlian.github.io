1. 分类问题

1.1 Precision

precision：预测为positive 的样本有n个，true positive有m个，precision = n / m = TP / (TP + FP)

1.2 Recall

recall：ground truth 中positive样本（应该预测为positive）有p 个，ground truth 中为positive且预测为positive的样本（true positive）有q个，recall = q / p = TP / (TP + FN)

1.3 mAP (mean Average Precision)

  • P：precision
  • AP：对每个recall对应的precision求均值（recall为横轴，precision为纵轴的recall-precision曲线下的面积）
  • mAP：m类标签的AP求均值

对一个二分类问题，假设有20个实例，它们的真实类别和分类模型给出的positive confident score 如下：

（2）计算AP (Average Precision)
AP是说，我们不是只通过top-5的precision 和 recall 衡量模型好坏，而是用从top-1到top-N的precision 和recall来衡量，显然随着样本增多，recall会越来越低，precision会整体呈下降趋势。
precision-recall曲线 (R曲线) 是以recall 为横坐标，precision 为纵坐标的曲线。

AP 计算有两种方法：

  • 设置一组阈值，比如 [0, 0.1, 0.2, …, 1]，对recall大于每一个阈值，都会得到一个对应的最大precision，这样就得到了11个precision，AP 就是这11个precision的平均值。

  • 假设 N 个样本中有M 个正例，每个正例对应的Recall就是1/M, 2/M, …, M/M 共 M 个recall值，对每个recall值，可以计算出对应的最大的precision， 对这M个precision求平均就是AP。

（3）计算 mAP
AP 衡量的是模型在单个类别上的好坏，mAP 是衡量在所有类别上的好坏。

1.4 ROC 和 AUC

ROC 关注两个指标：

  • True Positive Rate 将正例分对的比例：
  • False Positive Rate 负例中有多少分为了正例:

ROC 空间以FPR 为横坐标，TPR 为纵坐标，描绘的是模型在“正确分类的正例TP”和 “错分为正例的 FP ”之间的关系

在绘制 ROC 曲线时，confidence score是连续的，通过设置不同的阈值，把实例分为正类和负类，计算得到不同TPR 和 FPR，绘制ROC 曲线。

AUC 就是ROC 曲线和横坐标围成的面积，可以把AUC理解为一个概率值，随机挑选一个正样本和负样本，当前分类算法计算得到的score 将这个正样本排在负样本之前的概率，就是AUC 的值，AUC 越大越好。

希望AUC越大越好，0.5 < AUC < 1，AUC = 0.5相当于随机猜测，AUC < 0.5 比随机猜测差，但只要反着预测就优于随机猜测。

最优阈值就是找一个点，保证TPR高的同时FPR尽量小。找离(0,1)最近的点。

对于二分类问题，正负样本都足够的情况下用ROC。对多分类问题，用PR 曲线。
