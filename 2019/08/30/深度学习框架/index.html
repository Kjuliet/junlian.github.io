1. MegDL 全家桶

MegDL = MegBrain + MegSkull + MegHair + NeuPeak

  • MegBrain：类似 TensorFlow C++ core，数值计算的引擎，Engine
  • MegSkull：类似 TensorFlow Python封装，Cars
  • MegHair：类似 keras，pipeline to run model，Road
  • NeuPeak：类似 keras，Experiment Orchestration，得看地图

MegBrain

MegBrain Core 是用C++实现的，外面加了一层MegBrain Python Wrapper。

2. 用MegSkull定义模型

定义模型的“逻辑”，以及一部分运行时的行为，更关心“逻辑”：

  • 逻辑：输入长什么样，模型结构长什么样，进行了什么样的操作，Loss 怎么算
  • 运行时行为：做多卡训练时Input tensor 怎么样拆分到多卡上，即计算节点(如GPU)，什么时候把各个卡的结果汇总。

MegDL 图的基本概念

Tensor：网络中的一切数据，包括输入、参数和特征图，等价于数学上的概念；
Variable Node (VarNode)：包含 tensor 的节点，是数据的容器，被操作的对象；
Operator Node (OprNode)：

  • 网络中所有的动词，对数据进行的操作，比如conv，+，shape_of…
  • 一般有至少一个输入的 VarNode
  • 有至少一个输出的VarNode
    Computation Graph: 由 Operator Nodes 和 Variable Nodes 组成的网络。

基于图计算的模型逻辑：

MegBrain 图的特点：

  • 绝大多数 Operator 是无副作用的，没有内部状态的纯函数，无内部状态是指operator内部没有参数，比如FullyConneted有x, w, b三个外部输入，它的内部是没有参数的。但也有个别反例，如BatchNormalization, GaussianRNG
  • 网络的状态保存在参数 ParamProvider、常量 ConstProvider对应的Variable Node中；
  • Operator 具有前向计算规则和求导规则，可以自动求全图有关系的任意两点的导数。

一个模型从定义到运行经过了如下阶段：

  • 构建阶段：Operator 组合定义一个图，varnode 可以自动推导，一般不需要手动声明；
  • 编译阶段：选择若干个输出节点，编译计算图，涉及到图优化、分配device，分配内存；？？
  • 执行阶段：输入数据，计算输出

只有定义阶段是python 控制的，执行阶段python难以直接控制！

声明式 vs 指令式
MegDL 使用声明式逻辑，先定义好计算规则，最后输入数据，得到输出，类似于 theano, tf

指令式逻辑，给定输入数据，一步一变换，最后得到输出。如torch，pytorch，缺点是执行效率低，没有办法做预先的优化

网络中间变量均由以下三种数据容器经过 Operator 变换而成：

  • DataProvider：网络的输入节点，在声明时必须指定shape，求值时必须赋值，相当于 tf 的placeholder；
  • ParamProvider：网络参数的容器，训练时会使用优化器调整数值，相当于tf的Variable；
  • ConstProvider：网络中的常量

MegBrain默认使用(n, c, h, w) 的Layout，全世界只有tensorflow默认使用 (n, h, w, c)的Layout

3. MegDL 训练

    1. 定义网络结构
    • network = Network(outputs = [y], loss = loss)
    1. 生成训练数据
    • minibatch_generator()
    • yield {‘data’: }
    1. 训练
    • 图编译，把定义好的网络编译成一个可执行的function,每次把数据输进去的时候，它就会进行梯度下降，优化，device=’gpu0’，func_maker.make_func(loss_var = net.loss_var)；optimizer.Momentum()
    • 图执行：out=train_func(**minibatch)，从数据集里拿到数据，优化。
    1. 测试
    • 编译：classify = Function().compile(net.outputs[0])，建立一个Function对象，选择网络输出的点，complie一下，得到一个callable的function，然后把输入数据扔进去。
    • 执行：classify(np.array([0.5, 0.3]))
    • 编译并执行：pred = net.outputs; pred.eval(data=np.array([0.5, 0.3])) 得到一个varnode，通过eval()把数据喂进去也是可以的，其实eval()就是上面两句的语法行，每次调用都会编译执行前两部。

4. debug 技巧

4.1 常用的tensor 操作：

  • 基本运算：$+, -, , /, %, *, @$ @是矩阵乘
  • 逻辑运算：$>, <, \le, \ge, \ne, eq$ 比较实际值要用eq，==比较的是python对象
  • 索引：$[]$, set_sub, AdvancedIndexing
  • 统计信息：min, max, mean, sum, prod
  • 维度变换类：dimshuffle, reshape, add_axis, T, broadcast, shape

4.2 Tensor 的shape 和 partial_shape的区别：

  • 每个varnode x 都有partial_shape和shape两个属性；
  • x.partial_shape 是根据其依赖的DataProvider声明的尺寸推导得到的，是一个python 可读的数组；
  • x.shape是一个符号变量(varnode)，表示张量的实际尺寸，只有在执行阶段得到实际的输入数据时才有值；
  • MegBrain在运行时不检查输入尺寸，只要输入数据和网络变换在数学上是成立的就能运行，例如conv对图像空间大小不要求，对通道数量有要求；
  • 网络中应尽量使用shape，一般用partial_shape辅助调试；
  • 取batch_size时，因为多卡数据划分，必须用x.shape[0]

4.9 加入Weight Decay

错误用法：把weight的L2加入loss，但是这种方法在多卡训练时每张卡上的weight被重复计算，相当于weight_decay增倍。
正确方法：loss = O.WeightDecay(loss, {‘*:W’:1e-3})
