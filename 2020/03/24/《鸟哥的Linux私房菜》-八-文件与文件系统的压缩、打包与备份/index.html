文件里有很多并不是完全填满的空间，压缩就是将这些空间填满，让整个文件占用的容量下降，以此来提高磁盘利用率，在网络数据传输中解放更多带宽，有的压缩程序还可以进行容量限制，将大型文件分片，变成数个小型文件方便小存储设备携带。计算机系统都用 bytes 单位计量，但计算机最小的计量单位是 bits，1 bytes = 8 bits，如果记录数字 “1”，只会占最右边的 1 个bit，其他 7 个 bits 会自动填 0，通过一些复杂的计算方式将这些没有用到的空间丢出来，让文件占用的空间变小，就属于压缩技术。另外，还有一种压缩技术是将重复的数据进行统计，例如数据是 “1111…1111” 共 100 个 1， 那么压缩技术会记录为 “100 个 1”。

将压缩后的文件还原回来称为 “解压缩”，压缩后的文件与原文件占用磁盘空间大小的比例，称为 “压缩比”

1 压缩指令

Linux 不同压缩指令所用的压缩技术不同，所以解压缩前，需要知道该文件是哪种压缩指令制作的。虽然 Linux 的扩展名与文件的属性没有绝对关系，但为了方便判断还是加了参考性的扩展名，常见的有：

  • *.Z ：compress 程序压缩的文件。不常用；
  • *.zip ：zip 程序压缩的文件。由 windows 常用的 zip 来；
  • *.gz ：gzip 程序压缩的文件。由 GNU 开发，后来 GNU 由开发出了 bzip2 和 xz 这几个压缩比更好的压缩指令；
  • *.bz2 ：bzip2 程序压缩的文件；
  • *.xz ：xz 程序压缩的文件；
  • *.tar ：tar 程序打包的数据，并没有压缩过。通常上面的指令仅针对一个文件压缩解压缩，为了避免每次都要压缩解压缩一大堆文件，所以有了打包软件 tar，可以将很多文件甚至目录打包成一个文件；
  • *.tar.gz ：tar 程序打包的文件，且经过了 gzip 的压缩。GNU 将整个 tar 和压缩功能结合；
  • *.tar.bz2 ：tar 程序打包的文件，且经过了 bzip2 的压缩；
  • *.tar.xz ：tar 程序打包的文件，且经过了 xz 的压缩

compress 这个老老指令几乎没人用了，如果有 compress 建立出来的 .Z 文件，可以用 znew 将该文件转成 gzip 格式。

1.1 gzip

1.1.1 gzip 指令

gzip 是目前应用度最广的压缩指令，可以解压 compress、zip 和 gzip 等压缩过的文件，指令用法如下：
gzip [-cdtv#] 文件名

  • -c ：将原本要转成压缩文件的数据，变成文字类型从屏幕输出，可通过 > 做数据流重导向来处理，将原本应该由屏幕输出的数据，转成输出到文件而不是屏幕；
  • -d ：解压缩的参数；
  • -t ：检验压缩文件的一致性，查看文件是否有错误；
  • -v ：显示原文件，压缩文件的压缩比等信息；
  • -# ：# 为数字，代表压缩等级，-1 最快，但压缩比最差，-9 最慢，但压缩比最好，默认是 -6。

用 gzip 压缩后原文件被压缩文件替换后就不再存在了，同样用 gzip -d 解压缩后压缩文件就被原文件替换了。另外，用 gzip压缩的文件在 Windows 系统中，可以被 WinRAR/7zip 这个软件解压缩。

2 打包指令 tar

将多个文件或目录包成一个大文件的指令，称为“打包指令”，zip、bzip2 和 xz 在对目录进行压缩时，会将目录中所有文件“分别”进行压缩，而不是将所有文件包成一个文件。一般称仅打包的文件“tarfile”，打包加压缩的文件称为“tarball”。

2.1 打包与压缩

2.1.1 打包目录下所有文件

tar [-z|-j|-J][pPcv] [-f 待建立的打包文件名] 被打包文件名

  • -z ：通过 gzip 压缩，此时压缩文件名最好为 *.tar.gz；
  • -j ：通过 bzip2 压缩，压缩文件名最好为 *.tar.bz2；
  • -J ：通过 xz 压缩，压缩文件名最好为 *.tar.xz；
  • -p ：保留备份数据原本的权限和属性，常用于备份重要文件；
  • -P ：保留绝对路径，即允许备份数据中含有根目录存在；
  • -c ：建立打包文件，可搭配 -v 来察看过程中被打包的文件名；
  • -v ：将正在压缩处理的文件名显示出来；
  • -f ：tar 不会主动产生建立的打包文件名，需要自定义，扩展名很重要。f后要紧跟文件名，如果写成-jvfc filename 产生的文件名会变成 c。

3 系统备份

3.1 备份过程

一般系统比较重要的目录有：

  • /etc/ 配置文件
  • /home/ 用户的家目录
  • /var/spool/mail/ 系统中所有账号的邮件信箱
  • /var/spool/cron/ 所有账号的工作排成的配置文件

3.2 备份数据复原系统后的 SELinux 问题

如果要以备份数据回填到原本的系统中，必须特别注意复原后的系统的 SELinux 问题，尤其是在 /etc 等系统文件上。SELinux 是特别的细部权限设定，可能会让系统无法存取某些配置文件内容，影响系统的正常使用权限。

例如当用备份数据复原系统后发现无法正常登入，大部分原因是因为 /etc/shadow 这个密码文件的 SELinux 类型在还原时被更改了，系统的登入程序无法顺利存取。

有如下几种解决方式：

  • 在第一次复原系统后，不要立即重新启动，先用 restorecon -Rv /etc 自动修复一下 SELinux 类型；
  • 通过各种方式登入系统后，建立 /.autorelabel 文件，重启后会自动修复 SELinux 的类型，并会再次重启，之后就正常了；
  • 通过各种救援方式登入系统，修改 /etc/selinux/config 文件，将 SELinux 改成 permissive 模式，重启系统。

4 XFS 文件系统的备份与还原

tar 可以对目录树系统进行备份与还原，但不能对整个文件系统进行备份与还原。对 XFS 文件系统，可以用 xfsdump 和 xfsrestore 。

4.1 xfsdump 备份XFS 文件系统

xfsdump 可以进行文件系统的完整备份 (full backup) 和累积备份 (Incremental backup) ，第一次备份一定是完整备份，在 xfsdump 中定义为 level0，第二次备份时 level1 会比较当前文件与 level0 的差异，只备份变化过的文件，以此类推进行累积备份。各个 level 记录文件存放在 /var/lib/xfsdump/inventory 中。

xfsdump 使用限制：

  • 只能备份已挂载的文件系统；
  • 必须使用 root 权限；
  • 只能备份 XFS 文件系统；
  • 不支持特定目录的备份，如 /etc，因此 /etc 不是一个独立的文件系统；
  • 备份的数据只能被 xdsrestore 解析；
  • 不同备份两个具有相同 UUID 的文件系统，因为 xfsdump 是通过文件系统的 UUID 来分辨各个备份档的。

选项指令：
xfsdump [-L S_label] [-M M_label] [-l #] [-f 备份数据] 待备份数据

  • -L ：每次备份的 session 标头，可以填写针对此文件系统的简易说明；
  • -M ：存储媒体的标头，可以填写此媒体的简易说明。可以不加 -M 和 -L，但是会进入互动模式，要求 「enter」；
  • -l ：指定等级，有 0～9 共十个，预设为0，即完整备份；
  • -f ：产生的文件，可以是 /dev/st0 等装置文件名或其他一般文件名；

使用了 xfsdump 后会在 /var/lib/xfsdump/inventory 内产生文件，存储备份的相关信息，包括文件系统、时间、session ID 等，作为下次备份的一个参考依据。

xfsdump -I
-I ：用 -I 可以从 /var/lib/xfsdump/inventory 列目前备份的信息；

4.2 xfsrestore 还原XFS 文件系统备份

  1. 用 xfsrestore -I 查看 xfsdump 后的备份数据内容，和 xfsdump -I 一样，到 /var/lib/xfsdump/inventory/ 中找数据，查询重点是找出哪个文件是哪个挂载点，备份文件是什么 level 的，重点是 session label；

  2. 单一文件全系统复原 xfsrestore [-f 备份文件] [-L Session_label] [-s] 待复原目录，复原 level 0 的文件系统，将文件系统整个复原到最原本的状态，只要知道 level 0 的 session label 就可以。复原时原本待复原目录里面的东西没有被删除，同名的文件会被覆盖，其他文件会被保留。

  3. 通过累积备份文件复原系统 xfsrestore [-f 备份文件] -r 待复原目录，备份数据按 level 0 -> level 1 -> 顺序复原到系统内。

  4. 互动模式复原 xfsrestore [-f 备份文件] -i 待复原目录。如果想要仅还原部分数据，但根本不知道备份档里面有啥文件，或者要复原的文件数量太多时，可以通过 -i 进入互动界面：

5 光盘写入工具

归档常用 DVD/CD，通常先用 mkisofs 指令来将需要备份的数据建成一个映像文件 iso，然后用 ‘cdrecord’ 指令将该映像文件刻录到光盘或 DVD 中。

5.1 mkisofs 建立映像文件

5.1.1 建立不可开机的一般数据光盘映像文件

要想将数据刻录到 DVD 中，要先将数据报成一个映像文件，才可以写入 DVD 片中：
mkisofs [-o 映像文件] [-Jrv] [-V vol] [-m file] 待备份文件 -graft-point isodir=systemdir ...
-o ：生成的映像文件名；
-J ：生成比较兼容 windows 系统的文件名结构，可使文件名长度增加64个 unicode 字符；
-r ：通过 Rock Ridge 产生支持 Unix/Linux 的文件数据，可记录较多的信息，如 UID/GID等。一般数据光盘的格式为 iso9660，这种格式一般只支持旧版的DOS文件名，即文件名8个字符，扩展名3个字符，能够记录的信息很少；
-v ：显示建立 ISO 文件的过程；
-V ：建立 Volume，有点像 Windows 里看到的 C、D title 的东西；
-m ：排除文件，后面接的文件不备份到映像档中，可以使用通配符 *；
-graft-point ：转嫁或移植。一般默认所有要被加到映像文件中的文件都会被放置到映像文件的根目录，这样刻录后文件分类很不方便，而且出现命名相同的文件会导致有些数据无法被包含进来，用 -graft-point 选项通过 映像文件中的目录所在 = 实际 Linux 文件系统的目录所在 来定义位于映像文件中的目录，如 `-graft-point /linux/etc = /etc’ 表示将 /etc 内的所有数据备份到映像文件中的 /linux/etc 目录中；

6.2 cpio

cpio 可以备份任何东西，包括装置设备文件，但它不会主动去找文件来备份，所以一般来说需要配合类似 find 等可以找到文件名的指令来告诉 cpio 该备份的数据在哪，过程中涉及到数据流重导向，因为 cpio 会将整个数据显示在屏幕上，所以可以通过 > 将这些屏幕的数据重新导向到一个新的文件，还原就是将备份文件读进来 < 处理。

系统里的 /boot/initramfs-xxx 就是使用 cpio 建立的。

6.2.1 备份

cpio -ovcB > [filedevice]

  • -o ：将数据 copy 输出到文件或装置上；
  • -v ：显示存储过程中的文件名；
  • -c ：用一种较新的 portable format 方式存储；
  • -B ：预设 Blocks 是 512 bytes，可以增设到 5120 bytes，让大文件的存储速度加快；

6.2.2 还原

cpio -vcidu < [filedevice]

  • -i ：将数据从文件或装置 copy 到系统中；
  • -d ：自动建立目录，使用 cpio 备份的数据内容不一定会在同一层目录，因此要让 cpio 在还原时可以建立新目录；
  • -u ：自动用较新的文件覆盖较旧的文件；
